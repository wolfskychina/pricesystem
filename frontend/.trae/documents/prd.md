# PRD: 银行做市商报价监控面板

## 1. 产品概述

银行做市商交易系统的前端报价监控面板，用于实时展示交易所行情、做市商客户报价，并提供订单管理操作界面。

## 2. 目标用户

- 银行交易员：监控市场行情和客户报价，管理订单
- 风控人员：查看订单状态和风控结果

## 3. 功能需求

### 3.1 行情报价看板（核心功能）

- **实时行情表格**：展示所有合约的交易所行情和做市商报价
  - 字段：合约代码、交易所买价、交易所卖价、银行买价、银行卖价、中间价、价差(bps)、涨跌、涨跌幅、更新时间
  - 涨跌用颜色区分（红跌绿涨）
  - 价格格式化（保留2-4位小数）

- **刷新频率控制**：可切换 1s / 5s / 10s / 30s 自动刷新
  - 默认 5s
  - 支持手动暂停/恢复

### 3.2 订单管理

- **下单面板**：
  - 选择合约代码
  - 买卖方向（BUY/SELL）
  - 订单类型（MARKET/LIMIT）
  - 数量输入
  - 限价单需要输入价格
  - 提交下单

- **订单列表**：
  - 展示当前和历史订单
  - 字段：订单ID、合约、方向、数量、成交数量、价格、状态、时间
  - 状态标签颜色区分
  - 支持取消未成交订单

### 3.3 页面布局

- 顶部导航栏：系统名称、连接状态指示器
- 主区域：行情报价表格（占主要空间）
- 底部区域：下单面板 + 订单列表（标签页切换）

## 4. 非功能需求

- 响应式设计，支持 1280px+ 宽屏
- 页面加载时间 < 2s
- 刷新间隔精确，不漏数据
- 网络异常时显示错误提示，自动重试

## 5. 技术约束

- 技术栈：React 18 + TypeScript + Vite + Tailwind CSS
- 数据获取：REST API 轮询（后端服务端口：market-data 8082, pricing 8083, oms 8084）
- 不引入状态管理库，使用 React hooks 管理状态
- 不引入 UI 组件库，使用 Tailwind CSS 手写样式

## 6. API 接口

| 服务 | 方法 | 路径 | 说明 |
|---|---|---|---|
| market-data | GET | /marketdata | 获取所有行情 |
| market-data | GET | /marketdata/{symbol} | 获取单个行情 |
| pricing | GET | /quotes/{symbol} | 获取客户报价 |
| pricing | POST | /quotes/rfq | 请求报价 |
| oms | POST | /orders | 下单 |
| oms | GET | /orders | 查询订单列表 |
| oms | GET | /orders/{orderId} | 查询订单详情 |
| oms | DELETE | /orders/{orderId} | 撤单 |
